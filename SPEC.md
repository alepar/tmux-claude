# SPEC.md — tmux-claude

Full specification of the tmux setup for Claude Code workflows.

## Overview

A tmux configuration and set of shell scripts that provide rich visual feedback when running Claude Code sessions. Designed for workflows with multiple Claude instances running in parallel across tmux panes and windows.

## Features

### 1. Status Bar

**Left side**: Project-colored session badge
- 3-character wide colored bar followed by session name in bold white on dark grey
- Color is deterministically hashed from session name (consistent across restarts)
- 15-color palette: `204 114 039 220 183 209 156 081 141 215 117 168 149 075 229`
- Hash algorithm: `hash = (hash * 31 + charCode) % 65536`, then `hash % 15`

**Right side**: System stats and clock
- Format: `hostname │ CPU:XX% MEM:XX% CL:S%/W%/Son%/$Extra │ HH:MM TZ`
- CPU: sampled from `/proc/stat` (Linux) or `top -l` (macOS), 0.5s sample window. Red when >= 90%.
- MEM: from `/proc/meminfo` (Linux) or `vm_stat` (macOS). Red when >= 90%.
- CL: Claude usage from `api.anthropic.com/api/oauth/usage` — session%/weekly%/sonnet%/$extra
  - Authenticated via OAuth token from macOS Keychain or `~/.claude/.credentials.json`
  - Requires `User-Agent: claude-code/<version>` and `anthropic-beta: oauth-2025-04-20` headers
  - Cached for 60 seconds in `$TMPDIR/claude-usage-cache`
  - Falls back to `?` if credentials missing or API unreachable
  - Utilization percentages turn red at >= 90%, extra cost turns red at >= $150
- Refresh interval: 3 seconds (`status-interval 3`)

### 2. Window Tabs (Status Line)

**Inactive window tabs**:
- Normal: grey text `INDEX:NAME`
- Idle Claude: orange background, dark bold text `INDEX:NAME`
- Controlled by `@claude_idle` window option via `#{?@claude_idle,...}` conditional

**Current (active) window tab**:
- Always: white bold text on dark grey background `INDEX:NAME`
- Never shows orange highlight (user is already looking at it)

### 3. Pane Headers (Border Labels)

Each pane's top border displays a label generated by `pane-label.sh`.

**Normal state** (no idle Claude, or active pane):
- Active pane: blue pane number + git info
- Inactive pane: grey pane number + git info
- Git branch: blue (`colour075`)
- Git worktree: worktree directory name in orange (`colour208`)

**Idle state** (inactive pane with idle Claude):
- Dark text on orange background: `INDEX: GIT_LABEL`
- Active pane **never** shows idle highlight, even if marker exists

**Git detection**:
- Walks process tree from pane PID to deepest child
- For Claude/node processes: reads cwd from `$TMPDIR/claude-cwd%XX` marker
- For other processes: reads `/proc/PID/cwd` (Linux) or uses `lsof` (macOS)
- Detects worktree by comparing `git rev-parse --git-common-dir` vs `--git-dir`
- Shows `basename` of worktree toplevel (not the branch name)

### 4. Claude Idle Detection

**Definition of "idle"**: Claude Code is waiting for user input (not processing, not running tools).

**How it works**:

```
Claude finishes all work
  → Notification[idle_prompt] hook fires
    → touch $TMPDIR/claude-idle%XX
    → tmux set -w @claude_idle 1
      → pane header turns orange (if pane inactive)
      → window tab turns orange (if window inactive)

User submits next prompt
  → UserPromptSubmit hook fires
    → rm $TMPDIR/claude-idle%XX
    → tmux set -wu @claude_idle
      → highlighting clears

User focuses the pane/window
  → after-select-pane / after-select-window tmux hook
    → rm $TMPDIR/claude-idle%XX
    → tmux set -wu @claude_idle
      → highlighting clears (dismiss notification)
```

**Why `Notification[idle_prompt]` and NOT `Stop`**:
The `Stop` hook fires after every assistant response, including between tool calls. This means a Claude session running a 30-minute solver would show as "idle" the entire time. `Notification` with `idle_prompt` matcher only fires when Claude is genuinely waiting for user input.

**Why `@claude_idle` window option AND marker files**:
- `@claude_idle` drives the `#{?...}` conditional in `window-status-format` (tab highlighting)
- Marker files are read by `pane-label.sh` for per-pane header highlighting
- Both are needed because tmux format conditionals resolve at window scope, not pane scope

### 5. CWD Tracking for Claude

**Problem**: Claude Code's node process never calls `chdir()`. The Bash tool tracks CWD internally. So `/proc/PID/cwd` always shows the directory where Claude was launched, not where it's currently working.

**Solution**: `PostToolUse[Bash]` hook parses the command for `cd` invocations and writes the new directory to `$TMPDIR/claude-cwd%XX`. The `pane-label.sh` script reads this marker when the deepest process is `claude` or `node`.

**Regex**: `^[[:space:]]*cd[[:space:]]+(\"([^\"]+)\"|\'([^\']+)\'|([^\"\'[:space:]\&\;\|]+))`
- Matches: `cd /path`, `cd "/path with spaces"`, `cd '/path'`
- Only accepts absolute paths (`== /*`) that exist (`-d`)

### 6. Stale Marker Cleanup

Marker files persist after panes close. Cleanup runs on:
- `after-split-window`
- `after-new-window`
- `after-new-session`

The `cleanup-markers.sh` script:
1. Lists all live pane IDs via `tmux list-panes -a`
2. Scans `$TMPDIR/claude-idle%*` and `$TMPDIR/claude-cwd%*`
3. Removes any whose pane ID is not in the live list

### 7. Pane Border Styling

- Active pane: blue border line (`colour075`)
- Inactive pane: dark grey border line (`colour238`)
- Border labels appear at top (`pane-border-status top`)
- Single-pane windows still show the label (no special handling)

## tmux Configuration Details

### Base Settings
- `default-terminal "tmux-256color"` with RGB override
- Mouse enabled, 50k history, 1-based indexing
- Windows renumber on close, focus events enabled
- Zero escape time (no delay after Escape key)

### Hooks

| Hook | Trigger | Action |
|------|---------|--------|
| `after-select-pane` | User switches pane | Clear idle marker + `@claude_idle` for target pane |
| `after-select-window` | User switches window | Clear idle marker + `@claude_idle` for active pane |
| `after-split-window` | New pane created | Run `cleanup-markers.sh` |
| `after-new-window` | New window created | Run `cleanup-markers.sh` |
| `after-new-session` | New session created | Run `cleanup-markers.sh` |

**Note on `$TMPDIR` in tmux hooks**: tmux interprets `${VAR:-default}` as its own environment variable syntax and rejects the `:-` fallback operator. The workaround is `T=$TMPDIR; [ -z "$T" ] && T=/tmp`.

## Claude Code Hook Configuration

Added to `~/.claude/settings.json` under the `hooks` key:

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Bash",
      "hooks": [{"type": "command", "command": "bash ~/.tmux/claude-cwd-hook.sh"}]
    }],
    "Notification": [{
      "matcher": "idle_prompt",
      "hooks": [{"type": "command", "command": "[ -n \"$TMUX_PANE\" ] && touch \"${TMPDIR:-/tmp}/claude-idle${TMUX_PANE}\" && tmux set -w -t \"$TMUX_PANE\" @claude_idle 1"}]
    }],
    "UserPromptSubmit": [{
      "matcher": "",
      "hooks": [{"type": "command", "command": "[ -n \"$TMUX_PANE\" ] && rm -f \"${TMPDIR:-/tmp}/claude-idle${TMUX_PANE}\"; tmux set -wu -t \"$TMUX_PANE\" @claude_idle 2>/dev/null; true"}]
    }],
    "SessionStart": [{
      "matcher": "",
      "hooks": [{"type": "command", "command": "[ -n \"$TMUX_PANE\" ] && pwd > \"${TMPDIR:-/tmp}/claude-cwd${TMUX_PANE}\" && touch \"${TMPDIR:-/tmp}/claude-idle${TMUX_PANE}\" && tmux set -w -t \"$TMUX_PANE\" @claude_idle 1"}]
    }],
    "SessionEnd": [{
      "matcher": "",
      "hooks": [{"type": "command", "command": "[ -n \"$TMUX_PANE\" ] && rm -f \"${TMPDIR:-/tmp}/claude-idle${TMUX_PANE}\"; tmux set -wu -t \"$TMUX_PANE\" @claude_idle 2>/dev/null; true"}]
    }]
  }
}
```

The installer **merges** these hooks with any existing settings.json content (preserves other keys like `enabledPlugins`). It does not blindly overwrite.

## Claude Usage API

**Endpoint**: `GET https://api.anthropic.com/api/oauth/usage`

**Authentication**:
```
Authorization: Bearer <token from macOS Keychain or ~/.claude/.credentials.json → claudeAiOauth.accessToken>
Content-Type: application/json
User-Agent: claude-code/<version>
anthropic-beta: oauth-2025-04-20
```

**Credential sources** (tried in order):
1. macOS Keychain: `security find-generic-password -s "Claude Code-credentials" -w`
2. File: `~/.claude/.credentials.json`

**Note**: The `User-Agent` header must match `claude-code/<version>`. The API returns HTTP 429 without it.

**Response**:
```json
{
  "five_hour": {"utilization": 44.0, "resets_at": "2026-02-10T10:00:00Z"},
  "seven_day": {"utilization": 69.0, "resets_at": "2026-02-14T19:00:00Z"},
  "seven_day_oauth_apps": null,
  "seven_day_opus": null,
  "seven_day_sonnet": {"utilization": 45.0, "resets_at": "2026-02-14T19:00:00Z"},
  "seven_day_cowork": null,
  "iguana_necktie": null,
  "extra_usage": {"is_enabled": true, "monthly_limit": 5000, "used_credits": 1319.0, "utilization": 26.38}
}
```

The status bar displays `five_hour.utilization`/`seven_day.utilization`/`seven_day_sonnet.utilization`/`extra_usage.used_credits` (converted to dollars).

## Known Limitations

1. **tmux per-pane options leak to window scope**: `pane-border-style` set with `-p` affects all panes in the window in tmux 3.5. Workaround: marker files read by per-pane script.

2. **Format conditionals resolve at window scope**: `#{?@option,...}` in `pane-border-format` checks the window-level value, not per-pane. Workaround: `pane-label.sh` checks marker files per pane ID.

3. **`${VAR:-default}` not supported in tmux hooks**: tmux parses `${...}` as its own env var syntax. Workaround: `T=$VAR; [ -z "$T" ] && T=default`.

4. **Claude CWD tracking is regex-based**: Only catches `cd /absolute/path` patterns. Won't track `pushd`, `cd ~`, or relative paths. Sufficient for Claude Code's typical behavior (always uses absolute paths).

5. **Usage API requires OAuth login**: Only works with `claude.ai` OAuth authentication (the default for `claude` CLI). Won't work with API key auth.

6. **New Claude sessions start idle**: `SessionStart` hook sets idle state. Since the current window tab never shows orange, this is only visible if you start Claude in a background pane.
